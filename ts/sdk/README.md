# `@yield-adapter-standard/sdk`

The integrator SDK for the Solana Yield Adapter Standard. One client routes every position action
through the on-chain **dispatcher** (registry-gated, non-custodial); one decoder reads every
adapter's position; one account-builder per adapter is the only protocol-specific surface.

```
ts/sdk/
  client.ts        YieldAdapterClient — init / deposit / currentValue / withdraw / settle
  decode.ts        THE single decoder: decodePosition / decodeWithdrawalTicket / decodeAdapterEntry
  pdas.ts          position / vault_authority / ticket / registry / adapter-entry / ATA derivations
  returnData.ts    current_value() return-data → u64
  registry.ts      listAdapters / listActiveAdapters (enumerate AdapterEntry PDAs)
  conformance.ts   re-exported runConformance (self-certify a new adapter)
  adapters/        one AdapterDef per reference adapter (kamino, marginfi, jlp, maple, drift stand-in)
```

## One client, any adapter

```ts
import * as anchor from "@anchor-lang/core";
import { YieldAdapterClient, kaminoAdapter, usdc } from "./ts/sdk";

const client = new YieldAdapterClient({
  provider,                         // anchor.AnchorProvider
  dispatcher,                       // ya_dispatcher Program (from its IDL)
  adapter: kaminoProgram,           // the target adapter Program (from its IDL)
  def: kaminoAdapter,               // the adapter's account-builder
});

await client.initPosition();
await client.deposit(usdc(25));               // 25 USDC
const value = await client.currentValue();    // bigint — redeemable USDC (view, no tx)
const pos = await client.fetchPosition();     // decoded via the single decoder
await client.withdraw(pos!.shares);           // instant adapters settle here
```

Switching protocols is one import — swap `kaminoAdapter`/`kaminoProgram` for
`marginfiAdapter`, `jlpAdapter`, `mapleAdapter`, … The client, the dispatcher route, the decoder,
and the ticket lifecycle are identical for all of them.

## Two-phase withdrawals (cooldown adapters)

```ts
await client.withdraw(pos!.shares);           // opens a Pending WithdrawalTicket
const t = await client.fetchTicket();         // { status: "pending", unlockTs, ... }
// ...after the cooldown unlock...
await client.settle();                        // Pending → Settled
// or poll-then-settle in one call:
await client.waitAndSettle();
```

## The single decoder

`Position` and `WithdrawalTicket` are generated by `declare_ya_accounts!` **inside each adapter**, so
their field layout and 8-byte discriminators are byte-identical across all five reference adapters
(and any conformant third-party adapter). One decoder reads them all — no per-adapter codegen:

```ts
import { decodePosition } from "./ts/sdk";
const pos = decodePosition((await connection.getAccountInfo(positionPda))!.data);
// { owner, baseMint, adapter, shares, cachedValue, valueUpdatedTs, bump, vaultAuthorityBump }
```

Validated against all five live adapters on mainnet-fork (the conformance suite's *"single SDK
decoder reads this adapter's Position"* gate runs for every adapter) plus offline byte-vector tests.

## Discover what's registered

```ts
import { listActiveAdapters } from "./ts/sdk";
const active = await listActiveAdapters(connection); // [{ programId, baseMint, name, status, ... }]
```

## Self-certify a new adapter

```ts
import { runConformance } from "./ts/sdk";
describe("my-adapter", () => {
  runConformance(() => ({ /* label, adapter, baseMint, depositAmount, remaining-account builders, ... */ }));
});
```

## Tests / acceptance

- `npx tsc --noEmit -p tsconfig.json` — clean.
- `npx ts-mocha -p ./tsconfig.json tests/sdk/decode.spec.ts` — offline decoder byte-vectors.
- `bash scripts/fork-test.sh tests/sdk/e2e.spec.ts` — every live adapter, init→deposit→currentValue→
  decode→withdraw (and the two-phase request→cooldown→settle) through the client + the real
  dispatcher, against real cloned mainnet state.

## Stack

`@solana/web3.js` + `@anchor-lang/core` (matching the conformance harness the SDK reuses for its fork
round-trips). Surfnet cheatcodes (fork fixtures) are raw `fetch` and live in `tests/helpers`.
